# ⚡ APEX LABS // Intelligent Athletic Conditioning

**Apex Labs** is a full-stack, AI-driven athletic performance application. It generates customized, dynamic training protocols based on a user's real-time biometric telemetry, shifting away from generic templates to offer highly specific, domain-aware sports conditioning. 

Built with a decoupled React/Django architecture, it processes live user metrics through an orchestrated OpenAI pipeline to deliver safe, effective, and structurally predictable workout JSON payloads.

---

## 🚀 The Architecture

This application utilizes a **fully decoupled Client-Server architecture**:
* **Frontend (The Client):** A high-performance, reactive dashboard built with React (Vite), styled via Tailwind CSS, and animated using Framer Motion for buttery-smooth visual feedback. It has zero direct access to the database.
* **Backend (The API Server):** Powered by Django REST Framework (DRF), which exposes a secure API to manage state, enforce security rules, and orchestrate heavy computational AI logic.
* **AI Engine:** Direct pipeline integration with the OpenAI API via custom Python orchestration, enforcing strict JSON schemas for flawless frontend rendering.

---

## ✨ Core Features & UI Polish

* **10-Step Biometric Gradient UI:** A fully custom, spring-animated numeric dial that tracks physical readiness. As telemetry shifts from 1 to 10, the UI dynamically re-themes from a "Critical Recovery" red to an "Apex Readiness" neon green.
* **Premium Holographic Loading:** A custom, animated skeleton-loader that pulses with a "radar scanner" effect while the AI calculates the split, entirely eliminating visual wait-time lag.
* **Smart Tissue Guardrails:** The backend intelligently parses compromised muscle data. If a user flags a sore area (e.g., "Chest"), the AI is strictly commanded to bypass that muscle group and focus on opposing tissue (e.g., back rows) to prevent injury.
* **Fault-Tolerant Fallback Layer:** Built-in exception handling in the Django views. If the cloud AI encounters network latency, rate limits, or key verification blocks, the Python backend instantly deploys an adaptive local matrix so the user interface never breaks.

## ⚙️ Domain-Specific Engines

### ⚽ The Pre-Pitch Football Matrix
Designed for use *before* stepping onto the turf. Focuses purely on pitch sharpness, speed, and skill work, entirely avoiding medical rehab terminology.
* **Solo Sharpness Mode:** Generates engaging individual skill drills (e.g., cone dribbling matrices, wall passes).
* **Match Ignition Mode:** Generates a dynamic team pre-game warm-up routine (e.g., progressive acceleration runs, positional shadow play).

### 🏋️ The Accessible Strength Tracker
Replaces hardcore bodybuilding terminology with approachable, highly effective movements.
* Focuses on standard equipment found in any small or home gym (dumbbells, adjustable benches, pull-up bars).
* Scales volume and intensity directly via the 1-10 System Energy Index and validated sleep tracking constraints.

## 🗄️ Database & Models (`core/models.py`)

* **`AthleteProfile`:** A `OneToOneField` linking exactly one unique profile to one user, ensuring the AI strictly adheres to their overarching athletic domain (Football vs. Gym).
* **`DailyLog`:** The heart of the telemetry engine. A `ForeignKey` relationship allowing users to have multiple daily logs. It records fatigue, sleep, and captures `sore_muscles` as a text field to inform the AI's tissue guardrails.
* **`WorkoutPlan`:** Utilizes a Django `JSONField()`. Instead of creating complex, heavy relational tables for individual sets and reps, the entire structured JSON payload generated by OpenAI is saved directly into a single, highly flexible column.

---

## 🛣️ API & Data Flow (`core/views.py` & `serializers.py`)

1. **Secure Border Control:** The React client transmits a raw JSON payload (e.g., `{"intensity": 5, "profile": "football"}`) across the network.
2. **Serialization (`ModelSerializer`):** Django translates this raw text into a secure Python object. The overridden `create()` method automatically attaches the `request.user` token to ensure secure, multi-tenant data privacy.
3. **AI Orchestration (`@api_view`):** The data is injected into a strict **System Prompt**. This forces the OpenAI model to behave like an elite athletic coach and return predictable JSON arrays rather than casual chat paragraphs.
4. **Delivery:** The View converts the AI's response back into an optimized JSON package and serves it back to React.

## 💻 Local Setup & Installation

To run this application locally on a dual-server setup, follow these steps:

### 1. Clone the Repository
```bash
git clone [https://github.com/yourusername/apex-labs.git](https://github.com/yourusername/apex-labs.git)
cd apex-labs
```

### 2. Backend Initialization (Django)
Open a terminal in the `backend` directory:
```bash
# Create and activate a Python virtual environment
python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate

# Install core dependencies
pip install django djangorestframework openai python-dotenv django-cors-headers

# Set up secure environment variables
# Create a .env file in the root backend directory and add your key:
echo "OPENAI_API_KEY=sk-proj-your_actual_key_here" > .env

# Run migrations to build the database schema and boot the server
python manage.py migrate
python manage.py runserver
```

### 3. Frontend Initialization (React/Vite)
Open a second terminal in the `frontend` directory:
```bash
# Install Node modules
npm install

# Start the Vite development server
npm run dev
```

The application will be live and rendering at `http://localhost:5173`.
